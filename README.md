# Flood Games Game Bot

Отдельный Telegram-бот для игровой валюты `Угольки`.

## Что уже есть

- `/start` в ЛС с главным меню.
- Личный профиль: баланс, username, роль, статус, дата попадания в базу, титул.
- Лавка услуг с покупкой за Угольки.
- Ручные заявки на услуги для админского подтверждения.
- Топ пользователей по балансу.
- Чатовые команды через точку:
  - `.подарить @username 50`
  - `.мои угольки`
  - `.угольки @username`
  - `.жар`
  - `.промокод КОД`
  - `.разжечь @username 100`
  - `.пригасить @username 50`
  - `.админ`
- Админ-панель:
  - создание услуг по шагам;
  - включение и выключение услуг;
  - заявки на покупки;
  - создание промокодов;
  - статистика;
  - простая текстовая рассылка.

## Настройка

1. Скопируй `.env.example` в `.env`.
2. Заполни:
   - `BOT_TOKEN` или `GAME_BOT_TOKEN`
   - `OWNER_ID`
   - `ADMIN_IDS`, если есть дополнительные админы через запятую
   - `DEVELOPER_USERNAME`, если нужна кнопка помощи
3. Для команд с точкой в группе отключи privacy mode у бота через `@BotFather`:
   - `/setprivacy`
   - выбрать бота
   - `Disable`

## Запуск

```bash
npm install
npm run dev
```

Сборка:

```bash
npm run build
npm start
```

## Bothost

Для Bothost используй Node.js `Debian Slim` и включи собственный `Dockerfile`.

Минимальные переменные окружения:

- `BOT_TOKEN`
- `OWNER_ID`
- `ADMIN_IDS`, если нужны дополнительные админы
- `MAIN_CHAT_ID`, если нужно ограничить команды одним чатом
- `DEVELOPER_ID` и `DEVELOPER_USERNAME`, если нужен разработчик в админ-доступе и помощи

База SQLite хранится в `/app/data/game_bot.sqlite`. Эта папка не должна попадать в Git и сохраняется между деплоями на Bothost.

## Важно

`game_bot` не использует базу `join_bot` и не зависит от нее. Интеграцию можно добавить позже отдельным слоем.
